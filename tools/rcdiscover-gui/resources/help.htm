rc_discover

Discovery of rc_visard devices

Devices that are powered up and connected to the local network or directly to a computer can be found using the standard GigE Vision ® discovery mechanism. At startup of the rcdiscover-gui tool, all available rc_visard devices are listed with their name, serial number, current IP, and unique MAC address. The discovery tool finds all devices reachable with global broadcasts. Thus, devices located in different subnets than the computer may also be listed. An icon in the discovery tool indicates whether such devices are actually reachable via a web browser.

After successful discovery, a double click on the device row opens the Web GUI of the device in the default web browser of the operating system. To rescan the network, click Rerun Discovery.

Reset of a misconfigured rc_visard

Using the Reset rc_visard button in the discovery tool, one can reset misconfigured devices by following below instructions.

  1. The reset mechanism is available for two minutes after device startup only. Thus, a reboot of the rc_visard may be required before being able to reset the device.
  2. If the misconfigured rc_visard is still successfully detected by the discovery tool, one can select it from the rc-visard drop down menu. Otherwise, the rc_visard's MAC address, which is printed on the device, may be entered manually in the designated fields.
  3. By clicking one of the four buttons, specific modules can be reset:
    • Reset Parameters: Reset rc_visard parameters that are configurable via Web GUI (e.g., frame rate).
    • Reset GigE: Reset GigE Vision ® settings (network configuration and user defined name).
    • Reset All: Reset rc_visard parameters as well as GigE Vision ® parameters.
    • Switch Partitions: Allows to perform a rollback the the previous firmware.
  4. A successful reset is indicated by a white status LED followed by a device reboot. If no reaction is noticable, the two minutes time slot mentioned in step 1 may have closed, requiring another reboot. Moreover, repeating step 3 multiple times often is useful.
